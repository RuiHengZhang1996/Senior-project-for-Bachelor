As people living in the so-called “Information Age”, we have access to an immense number of resources at our disposal. And most of us happen to use this to find what we need.

Everything from diagnosing your own disease to American politics to cute cat videos, it can be found with a click of a button. Now imagine looking for a cute cat video without Google. Or looking for a keyword in this.



Log File jungle exploring.


And the problem that we face now is exactly what Cisco ran into when changing their storage system for these files.

While technologies exist for log file searching, it defeated Cisco’s purpose of moving to Amazon’s storage system. Cost. Amazon’s search system called Athena charges around $5 per each click. Imagine paying $5 for every Google search you make!Not only is it not cost efficient, it is slow and unorganized. New data is coming in constantly and Athena can’t keep up. This means every time a new log file comes in, Athena needs to update, which makes the search slow.



Meet Our Team



WE ARE STORYTELLERS




We create Lambdalog!

We knew we could create something with the basic functionality of Athena, but make it more cost-effective and specialize it to log files and organize the storage.

It has a simple box to enter a keyword and an option to specify the type of logs to search and the date and time. After running a query, the application presents a table of the search results separated by different sections.



Why choose Lambdalog?


LambdaLog is cost efficient because it utilizes the Amazon service Lambda, which is free for the first million queries per month. Also, LambdaLog monitors new data coming into the storage system so that updates are not required.
